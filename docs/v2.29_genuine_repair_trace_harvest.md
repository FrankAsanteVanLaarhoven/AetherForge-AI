# v2.29 — Genuine Repair Trace Harvest

Milestone type: **trace harvest / dataset infrastructure**. **Not a training milestone** — no model
trained, no SWE-bench, no new model families, no weight/champion/index changes.

Branch: `feature/v229-genuine-repair-trace-harvest` (based on the pushed v2.28).

## Goal

v2.28 found the substrate had **0** genuine broken→fixed repair transitions (the model never
repaired; degenerate claimed-repairs were filtered). v2.29 produces genuine, execution-verified,
verifier-labelled FORMAT-repair transitions: `candidate fails → structured verifier signal → repair
plan → changed solution → verifier-pass`.

## Method (no held-out contamination)

`scripts/build_v229_repair_harvest.py` takes each known-correct tree serializer (the 12 v2.26
representation references), applies a controlled **output-format-only** perturbation, and keeps the
pair only when execution confirms the perturbed candidate FAILS and the canonical version PASSES.
The repair is linked to the v2.27 structured format verifier diagnosis.

- Perturbations (format only; algorithm preserved): separator, bracket type, missing/extra
  structural marker, string/list and tuple/list container mismatch.
- A candidate is kept only if it produces a wrong OUTPUT (no raise) that the v2.27 verifier
  classifies as a FORMAT failure type — so this harness and the v2.28 builder agree on the label.
- Each record: `candidate_solution(fail)`, `candidate_status`, structured `verifier_signal`
  (`failure_type/expected/observed/diagnosis/repair_hint`), `repair_plan` (verifier-linked),
  `repaired_solution`, `final_status(pass)`, `candidate_differs_from_final`, `repair_successful`,
  computed `contamination_guard`.

Rejected if: `candidate == final`; verifier signal missing; final status missing; contamination
overlap > 0; repair does not change output; repair fails; repair not linked to verifier diagnosis.

Output is LOCAL-ONLY / gitignored: `data/generated/v229/repair_traces.jsonl`,
`harvest_aggregate.json`.

## Result

- Records harvested: **9** (21 attempts; 12 dropped as ambiguous/raised-before-output).

- Failure-type distribution: type_error 4, extra_null_marker 2, missing_null_marker 1,

- Contamination guard violations: **0**.
