## Method (no held-out contamination)

`scripts/build_v230_broadened_repair_harvest.py` defines a library of NEW, non-held-out,
execution-verified reference functions (10 format-family + 10 algorithmic; names disjoint from the
46 benchmark callables) and applies **typed mutators** â€” each injects a KNOWN fault with a KNOWN
verifier `failure_type`. A pair is kept only when execution confirms the perturbed candidate FAILS
its asserts and the canonical version PASSES. The structured verifier signal carries the injected
`failure_type` plus the REAL executed `observed`/`expected` values.

- Algorithmic faults: off-by-one, operator flip, wrong base case, wrong init/return, wrong order â€”
  labelled `algorithmic_error` â†’ `algorithmic_repair`.
